# 在庫引当

## 機能概要

在庫引当機能は、取込済み受注データに対して商品別の引当可能数を確認し、出荷可能な明細を確定する機能である。

## 前提条件

- 受注ファイル取込が正常に完了していること。
- 商品マスタに対象 SKU が登録されていること。
- 在庫テーブルが当日分に更新済みであること。

## 処理仕様

### 引当対象抽出

| 条件 | 内容 |
| --- | --- |
| 受注ステータス | `IMPORTED` |
| エラー有無 | エラーなし |
| 販売可否 | 商品マスタが販売可 |
| 引当済み | 未引当 |

### 引当判定

```ts
const allocatableQuantity =
  stockQuantity - reservedQuantity - safetyStockQuantity;

const canAllocate = allocatableQuantity >= orderQuantity;
```

| 判定 | 条件 | 更新内容 |
| --- | --- | --- |
| 引当可 | 引当可能数 >= 注文数量 | 受注ステータスを `ALLOCATED` に更新 |
| 引当不可 | 引当可能数 < 注文数量 | 受注ステータスを `ALLOCATION_FAILED` に更新 |
| 対象外 | 販売不可商品 | エラーコード `E101` を設定 |

### 排他制御

同一商品に対する同時引当を避けるため、在庫テーブルの対象レコードを更新ロックする。ロック取得に失敗した場合は処理を中断し、リトライ対象としてバッチ実行履歴へ記録する。

## アクセスするデータ

| データ名 | アクセス | 用途 |
| --- | --- | --- |
| 受注テーブル | 参照、更新 | 引当対象取得、ステータス更新 |
| 在庫テーブル | 参照、更新 | 引当可能数確認、引当数更新 |
| 商品マスタ | 参照 | 販売可否、温度帯確認 |
| 引当履歴テーブル | 登録 | 引当結果の監査記録 |
