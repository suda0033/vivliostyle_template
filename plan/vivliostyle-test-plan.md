# Vivliostyle PDF化テスト計画

## 目的

AI駆動開発で作成した Markdown ドキュメントを、提出用 PDF として整った体裁で出力できるかを Vivliostyle で検証する。

検証では、Markdown から PDF 化する際に崩れやすい要素を一通り含め、本文、図表、コード、ページ構成、目次、脚注、リンク、スタイル指定が提出物として実用に耐えるか確認する。

## 想定する提出ドキュメント

- 開発計画書
- 要件定義書
- 設計書
- 実装方針書
- テスト計画書
- 調査レポート
- 生成AIとの対話ログを整理した技術メモ

## テストすべき機能

### 1. 基本的な Markdown 表現

- 見出しレベル H1 から H4
- 段落
- 太字、斜体、インラインコード
- 箇条書き
- 番号付きリスト
- ネストしたリスト
- 引用
- 水平線

確認観点:

- 見出しの階層が PDF 上で読みやすいか
- リストのインデントが崩れないか
- 長い段落でも行間と余白が適切か

### 2. ページ構成

- 表紙
- 改ページ
- 章ごとの開始位置
- ヘッダー
- フッター
- ページ番号
- 用紙サイズ
- 余白

確認観点:

- 表紙にページ番号を出さない、または本文と別扱いにできるか
- 章の途中で不自然な改ページが起きないか
- ページ番号やフッターが本文と重ならないか

### 3. 目次と参照

- 目次
- 見出しへの内部リンク
- 外部 URL リンク
- 図表番号
- 相互参照に近い表現

確認観点:

- PDF 内リンクが機能するか
- 目次の階層、ページ番号、折り返しが自然か
- 長い見出しでも目次が破綻しないか

### 4. コードブロック

- 短いコード
- 長いコード
- 横に長いコード
- 複数言語のコードブロック
- コマンド例
- JSON、YAML、Markdown の例

確認観点:

- 等幅フォントで読みやすいか
- 横に長い行がはみ出さないか
- ページをまたぐコードブロックが読みにくくならないか
- 背景色や罫線が印刷時にも見やすいか

### 5. 表

- 小さい表
- 横に長い表
- セル内で改行する表
- 数値、ステータス、備考を含む表

確認観点:

- 列幅が適切か
- ページ幅を超えないか
- セル内の文章が読める形で折り返されるか
- ヘッダー行が見やすいか

### 6. 画像と図

- ローカル画像
- キャプション付き画像
- 幅指定した画像
- スクリーンショット

確認観点:

- 画像が PDF に埋め込まれるか
- 画像の解像度が提出用として十分か
- ページからはみ出さないか
- 図の前後の余白が自然か

### 7. Mermaid 記法

- フローチャート
- シーケンス図
- クラス図
- 状態遷移図
- ER 図
- ガントチャート
- Git graph
- Markdown 内に直接書いた Mermaid ブロック
- Mermaid CLI などで SVG または PNG に変換した図の貼り込み

確認観点:

- Vivliostyle のビルド工程で Mermaid を扱えるか
- 直接記述した Mermaid ブロックを PDF 化できるか
- 事前変換した SVG または PNG がきれいに埋め込まれるか
- 図中の日本語が文字化けしないか
- 図がページ幅からはみ出さないか
- 図がページ分割されず、読みやすい位置に配置されるか
- 背景色、線、文字色が印刷時に見やすいか
- Mermaid のテーマを提出用 CSS と合わせられるか

### 8. 注釈と補足情報

- 脚注
- 補足コラム
- 注意書き
- TODO や NOTE のような開発メモ

確認観点:

- 脚注が正しいページに表示されるか
- 補足情報が本文と視覚的に区別できるか
- 提出物として不要な開発メモを目立たせる、または除外できるか

### 9. 日本語組版

- 日本語の長文
- 英数字混じりの文章
- URL を含む文章
- 括弧、句読点、記号
- 長いカタカナ語
- 禁則処理

確認観点:

- 日本語の改行位置が不自然でないか
- URL や長い英単語がページ外へはみ出さないか
- 和文と英文のフォントバランスが良いか

### 10. 提出用の体裁

- タイトル
- 作成日
- 作成者
- 版数
- 変更履歴
- 承認欄、確認欄

確認観点:

- ビジネス文書として必要な情報を自然に配置できるか
- 変更履歴表が読みやすいか
- 表紙、本文、付録の見た目に一貫性があるか

### 11. CSS カスタマイズ

- フォント指定
- 見出しデザイン
- 表のスタイル
- コードブロックのスタイル
- ページマージン
- `@page` 指定
- 印刷向けの色指定

確認観点:

- Vivliostyle の CSS Paged Media 指定が期待どおり効くか
- Markdown 由来の HTML 要素に安定してスタイルを当てられるか
- PDF 出力時に画面表示との差が許容範囲か

- `docs/vivliostyle-sample.md`
- `styles/document.css`
- `vivliostyle.config.js`

## 合格基準

- Markdown から PDF を生成できる
- 日本語文書として読みやすい
- ページ番号、見出し、表、コード、画像、Mermaid 図が崩れない
- 提出用の表紙と変更履歴を含められる
- CSS を調整して文書テンプレートとして再利用できる
- AI駆動開発で日常的に作る Markdown を大きく書き換えずに PDF 化できる
